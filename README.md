# 🛒 Retail Customer Behavior Analytics Using SQL

## 📌 Project Overview

This project focuses on analyzing retail sales data using SQL to uncover customer purchasing behavior, revenue trends, and category performance.

The analysis includes data exploration, customer insights, revenue analysis, and business-focused SQL queries to generate actionable insights from transactional retail data.

---

## 🎯 Objectives

* Analyze customer purchasing behavior
* Identify top-performing product categories
* Measure revenue performance
* Explore customer demographics
* Practice SQL concepts used in data analytics

---

## 🗂 Dataset Information

The dataset contains retail transaction records with the following fields:

* Transaction ID
* Sale Date
* Sale Time
* Customer ID
* Gender
* Age
* Product Category
* Quantity
* Price Per Unit
* COGS
* Total Sale

---

## 🛠 SQL Skills Used

* SELECT
* DISTINCT
* COUNT()
* SUM()
* AVG()
* GROUP BY
* ORDER BY
* LIMIT
* CASE WHEN
* HAVING

---

## 📊 Business Questions Solved

1. Total number of transactions
2. Total revenue generated
3. Average sale value
4. Unique customer count
5. Unique product categories
6. Revenue by category
7. Highest revenue category
8. Transactions by category
9. Revenue by gender
10. Average age by category
11. Revenue by age group
12. Top 5 customers by spending
13. Category with highest quantity sold
14. Quantity sold by category
15. Average quantity by category
16. Customer with highest spending
17. Categories with revenue greater than 150,000
18. Categories with more than 600 transactions
19. Genders with average revenue greater than 450

---

## 📈 Key Insights

* Revenue distribution varies across product categories.
* Customer spending behavior differs by age group.
* A small number of customers contribute significantly to total revenue.
* Product categories show different sales and quantity patterns.
* Gender-based purchasing trends can be identified through revenue analysis.
